# i-game-pages

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放和展示各类独立的 HTML 页面。每个页面均为自包含的完整网页，不依赖外部域名或特定网站环境。这些页面可能包含小游戏、工具页面、原型演示或其他静态网页内容。

## 目录说明

```
i-game-pages/
├── pages/          # 存放所有独立 HTML 页面
│   └── ...         # 每个页面以单独文件夹或文件形式存在
├── assets/         # 公共资源文件（图片、样式、脚本等）
├── index.html      # 页面索引（可选）
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹或文件对应一个独立页面，内部包含完整的 HTML、CSS、JavaScript 及相关资源。
- `assets/` 用于存放多个页面共享的静态资源，避免重复。
- 根目录的 `index.html` 可作为导航页，方便浏览所有已归档页面。

## 页面归档说明

每个归档的 HTML 页面应满足以下基本要求：

1. **完整性**：页面应自包含，所有资源尽量内联或存放于本仓库中，避免引用外部不可控资源。
2. **独立性**：页面之间互不依赖，可单独打开运行。
3. **文档化**：建议在页面文件夹内附带简单的说明文件（如 `README.md`），描述页面功能、操作方式或注意事项。
4. **兼容性**：尽量兼容现代主流浏览器，如需特定环境请注明。

## 维护说明

- 欢迎通过 Pull Request 提交新的独立页面，或对现有页面进行改进。
- 提交前请确保页面功能正常，无外部依赖，并符合上述归档要求。
- 如发现页面无法正常显示或存在明显问题，可提交 Issue 反馈。
- 仓库维护者会定期检查页面状态，清理失效或不符合要求的条目。
